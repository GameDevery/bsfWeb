Clips the provided 2D vertices to the provided clip rectangle.

The vertices must form axis aligned quads.

Parameters
[in,out]verticesPointer to the start of the buffer containing vertex positions.
[in,out]uvPointer to the start of the buffer containing UV coordinates.
[in]numQuadsNumber of quads in the provided buffer pointers.
[in]vertStrideNumber of bytes to skip when going to the next vertex. This assumes both position and uv coordinates have the same stride (as they are likely pointing to the same buffer).
[in]clipRectRectangle to clip the geometry to.

Clips the provided 2D vertices to the provided clip rectangle.

The vertices can be arbitrary triangles.

Parameters
[in]verticesPointer to the start of the buffer containing vertex positions.
[in]uvPointer to the start of the buffer containing UV coordinates. Can be null if UV is not needed.
[in]numTrisNumber of triangles in the provided buffer pointers.
[in]vertStrideNumber of bytes to skip when going to the next vertex. This assumes both position and uv coordinates have the same stride (as they are likely pointing to the same buffer).
[in]clipRectRectangle to clip the geometry to.
[in]writeCallbackCallback that will be triggered when clipped vertices and UV coordinates are generated and need to be stored. Vertices are always generate in tuples of three, forming a single triangle.

Fill the pre-allocated vertex, uv and index buffers with the mesh data for the specified render element.

Parameters
[out]verticesPreviously allocated buffer where to store the vertices.
[out]uvPreviously allocated buffer where to store the uv coordinates.
[out]indicesPreviously allocated buffer where to store the indices.
[in]vertexOffsetAt which vertex should the method start filling the buffer.
[in]indexOffsetAt which index should the method start filling the buffer.
[in]maxNumVertsTotal number of vertices the buffers were allocated for. Used only for memory safety.
[in]maxNumIndicesTotal number of indices the buffers were allocated for. Used only for memory safety.
[in]vertexStrideNumber of bytes between of vertices in the provided vertex and uv data.
[in]indexStrideNumber of bytes between two indexes in the provided index data.
[in]renderElementIdxZero-based index of the render element.
[in]offsetPosition offset to apply to all vertices, after clipping.
[in]clipRectRectangle to clip the vertices to.
[in]clipShould the vertices be clipped to the provided clipRect.
