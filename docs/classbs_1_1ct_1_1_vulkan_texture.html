◆ copyImpl()

void copyImpl ( const SPtr< Texture > &  target,
const TEXTURE_COPY_DESC &  desc,
const SPtr< CommandBuffer > &  commandBuffer 
)
overrideprotectedvirtual

Copies the contents a subresource in this texture to another texture.

Texture format and size of the subresource must match.

You are allowed to copy from a multisampled to non-multisampled surface, which will resolve the multisampled surface before copying.

Parameters
[in]targetTexture that contains the destination subresource.
[in]descStructure used for customizing the copy operation.
[in]commandBufferCommand buffer to queue the copy operation on. If null, main command buffer is used.

Implements Texture.

Locks the buffer for reading or writing.

Parameters
[in]optionsOptions for controlling what you may do with the locked data.
[in]mipLevel(optional) Mipmap level to lock.
[in]face(optional) Texture face to lock.
[in]deviceIdxIndex of the device whose memory to map. If the buffer doesn't exist on this device, the method returns null.
[in]queueIdxDevice queue to perform the read/write operations on. See Device queues.
Note

If you are just reading or writing one block of data use readData()/writeData() methods as they can be much faster in certain situations.

Implements Texture.
