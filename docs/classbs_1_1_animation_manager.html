AnimationManager Class Reference
Internals » Core » Animation

Description

Keeps track of all active animations, queues animation thread tasks and synchronizes data between simulation, core and animation threads.

Determines how often to evaluate animations.

If rendering is not running at adequate framerate the animation could end up being evaluated less times than specified here.

Parameters
[in]fpsNumber of frames per second to evaluate the animation. Default is 60.

◆ update()

const EvaluatedAnimationData* update ( bool  async = true)

Evaluates animations for all animated objects, and returns the evaluated skeleton bone poses and morph shape meshes that can be passed along to the renderer.

Parameters
[in]asyncIf true the method returns immediately while the animation gets evaluated in the background. The returned evaluated data will be the data from the previous frame. Therefore note that this introduces a one frame latency on the animation. If the latency is not acceptable set this to false, at a potential performance impact.
Returns
Evaluated animation data for this frame (if async is false), or the previous frame (if async is true). Note that the system re-uses the returned buffers, and the returned buffer should stop being used after every second call to update(). This is enough to have one buffer be processed by the core thread, one queued for future rendering and one that's being written to.
