Collider Class Reference
Layers » Core » Physics

Description

Collider represents physics geometry that can be in multiple states:

  • Default: Static geometry that physics objects can collide with.
  • Trigger: Static geometry that can't be collided with but will report touch events.
  • Dynamic: Dynamic geometry that is a part of a Rigidbody. A set of colliders defines the shape of the parent rigidbody.

Inherited by BoxCollider, CapsuleCollider, MeshCollider, PlaneCollider, and SphereCollider.

◆ getContactOffset()

float getContactOffset ( )

Determines how far apart do two shapes need to be away from each other before the physics runtime starts generating repelling impulse for them.


This distance will be the sum of contact offsets of the two interacting objects. If objects are moving fast you can increase this value to start generating the impulse earlier and potentially prevent the objects from interpenetrating. This value is in meters. Must be positive and greater than rest offset.

Also see setRestOffset().
