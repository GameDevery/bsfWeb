AudioClip Class Referenceabstract
Layers » Core » Audio

Description

Audio clip stores audio data in a compressed or uncompressed format.

Clips can be provided to audio sources or other audio methods to be played.

Inherits Resource.

Inherited by FMODAudioClip, and OAAudioClip.

static HAudioClip create ( const SPtr< DataStream > &  samples,
UINT32  streamSize,
UINT32  numSamples,
const AUDIO_CLIP_DESC &  desc 
)
static

Creates a new AudioClip and populates it with provided samples.

Parameters
[in]samplesData streams containing the samples to load. Data will be read starting from the current position in the stream. The samples should be in audio format as specified in the desc parameter. Ownership of the data stream is taken by the audio clip and the caller must not close it manually.
[in]streamSizeNumber of bytes to read from the samples stream.
[in]numSamplesTotal number of samples (including all channels).
[in]descDescriptor containing meta-data for the provided samples.
Note
If the provided samples are in PCM format, they should be signed integers of provided bit depth.

◆ isCompressible()

bool isCompressible ( ) const
overrideprotectedvirtual

Returns true if the resource can be compressed using a generic compression when saved on a storage device.

Certain resources already have their contents compressed (like audio files) and will not benefit from further compression. Resources supporting streaming should never be compressed, instead such resources can handle compression/decompression locally through their streams.
