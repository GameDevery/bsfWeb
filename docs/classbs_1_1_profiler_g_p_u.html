Method documentation

◆ beginFrame()

void beginFrame ( )

Signals a start of a new frame.

Every frame will generate a separate profiling report. This call must be followed by endFrame(), and any sampling operations must happen between beginFrame() and endFrame().

◆ beginSample()

void beginSample ( const ProfilerString &  name)

Begins sample measurement.

Must be followed by endSample().

Parameters
[in]nameUnique name for the sample you can later use to find the sampling data.
Note
Must be called between beginFrame()/endFrame() calls.

◆ endFrame()

void endFrame ( )

Signals an end of the currently sampled frame.

Results of the sampling will be available once getNumAvailableReports increments. This may take a while as the sampling is scheduled on the core thread and on the GPU.

◆ endSample()

void endSample ( const ProfilerString &  name)

Ends sample measurement.

Parameters
[in]nameUnique name for the sample.
Note
Unique name is primarily needed to more easily identify mismatched begin/end sample pairs. Otherwise the name in beginSample() would be enough. Must be called between beginFrame()/endFrame() calls.

◆ getNextReport()

GPUProfilerReport getNextReport ( )

Gets the oldest report available and removes it from the internal list.

Throws an exception if no reports are available.

Note
Thread safe.

◆ getNumAvailableReports()

UINT32 getNumAvailableReports ( )

Returns number of profiling reports that are ready but haven't been retrieved yet.

Note
There is an internal limit of maximum number of available reports, where oldest ones will get deleted so make sure to call this often if you don't want to miss some.
Thread safe.
