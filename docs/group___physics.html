◆ CharacterClimbingMode

enum CharacterClimbingMode
strong

Controls climbing behaviour for a capsule character controller.

Normally the character controller will not automatically climb when heights are greater than the assigned step offset. However due to the shape of the capsule it might automatically climb over slightly larger heights than assigned step offsets.

Enumerator
Normal 

Normal behaviour.

Capsule character controller will be able to auto-step even above the step offset.

Constrained 

The system will attempt to limit auto-step to the provided step offset and no higher.

◆ D6JointDriveType

enum D6JointDriveType
strong

Type of drives that can be used for moving or rotating bodies attached to the joint.

Enumerator
X 

Linear movement on the X axis using the linear drive model.

Y 

Linear movement on the Y axis using the linear drive model.

Z 

Linear movement on the Z axis using the linear drive model.

Swing 

Rotation around the Y axis using the twist/swing angular drive model.

Should not be used together with SLERP mode.

Twist 

Rotation around the Z axis using the twist/swing angular drive model.

Should not be used together with SLERP mode.

SLERP 

Rotation using spherical linear interpolation.

Uses the SLERP angular drive mode which performs rotation by interpolating the quaternion values directly over the shortest path (applies to all three axes, which they all must be unlocked).

◆ PhysicsFlag

enum PhysicsFlag
strong

Flags for controlling physics behaviour globally.

Enumerator
CCT_OverlapRecovery 

Automatically recovers character controllers that are interpenetrating geometry.

This can happen if a controller is spawned or teleported into geometry, its size/rotation is changed so it penetrates geometry, or simply because of numerical imprecision.

CCT_PreciseSweeps 

Performs more accurate sweeps when moving the character controller, making it less likely to interpenetrate geometry.

When overlap recovery is turned on you can consider turning this off as it can compensate for the less precise sweeps.

CCT_Tesselation 

Large triangles can cause problems with character controller collision.

If this option is enabled the triangles larger than a certain size will be automatically tesselated into smaller triangles, in order to help with precision.

See also
Physics::getMaxTesselationEdgeLength
CCD_Enable 

Enables continous collision detection.

This will prevent fast-moving objects from tunneling through each other. You must also enable CCD for individual Rigidbodies. This option can have a significant performance impact.
